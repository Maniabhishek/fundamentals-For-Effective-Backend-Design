## TCP
- it requries to form a connection , which is Three steps TCP connection
- after connection , communication is lossless protocol, ensures by receving ack, until then it will keep sending the data packet
- confession control (if network is choked with high traffic, tcp delays the packet so as not to overwhelm the network)
- lot of overhead involved to maintain data accuracy

## UDP
- no connection required
- public IP address requried for Peer to peer connection
- lossy protocol (send each packet only once)
- smaller headers no extra overhead
